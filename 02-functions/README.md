## 📌 Overview

Functions are one of the core building blocks of JavaScript.
They help in writing reusable, structured, and maintainable code.

JavaScript executes code line by line, but functions run only when they are called.

---

## 🎯 Why Functions Matter

* Reduce code duplication
* Improve readability
* Make logic reusable
* Break complex problems into smaller parts

---

## 🧩 Types of Functions

### ▸ Function Declaration

Defined using the `function` keyword. These are hoisted.

### ▸ Function Expression

Stored inside a variable. Not accessible before initialization.

### ▸ Arrow Function (ES6)

Short syntax, widely used in modern JavaScript.

### ▸ Anonymous Function

Function without a name, mostly used in callbacks.

---

## 🔁 Parameters vs Arguments

* **Parameter** → variable in function definition
* **Argument** → value passed during function call

---

## ⚙️ Default Parameters

Used when no argument is provided to avoid `undefined` values.

---

## 📦 Rest Operator (`...`)

Collects multiple values into an array when number of inputs is unknown.

---

## 🔙 Return Values

`return` sends a value back and stops function execution.

---

## 🧠 First-Class Functions

Functions behave like values:

* Stored in variables
* Passed as arguments
* Returned from other functions

---

## 🔄 Higher Order Functions

A function that accepts or returns another function.

---

## ⚖️ Pure vs Impure Functions

### ✅ Pure

* Same input → same output
* No side effects

### ❌ Impure

* Depends on external variables
* Output may vary

---

## 🔒 Closures

Closures allow a function to access outer variables even after execution.

Used for:

* Data privacy
* State management

---

## ⚡ IIFE (Immediately Invoked Function Expression)

Runs immediately after definition.

Used to:

* Avoid global scope pollution
* Execute one-time logic

---

## 🧬 Hoisting

* Function Declaration → hoisted
* Function Expression → not usable before initialization
